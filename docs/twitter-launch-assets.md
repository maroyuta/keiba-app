# SNS立ち上げアセット集(X + TikTok)

作成: 2026-07-16。[twitter-strategy.md](twitter-strategy.md)の実行編。
文面はすべてコピペで使える状態。実績数値は`/dashboard`が溜まり次第差し替える。

**前提(できること/できないこと)**: アカウントの改名・ログイン・投稿の実行は
セキュリティ上AI側からはできない。代わりに「コンテンツ一式(画像・動画・投稿文)を
コマンド1発で生成→あなたは貼って投稿するだけ」の体制を実装済み(下記ルーティン参照)。

---

## 1. アカウント名

**推奨: 「AI競馬アナリスト」**(画像テンプレにも既に反映済み)

- 調査の結果、「競馬アナリスト」を固有名として使う大手アカウントは見当たらない(肩書きとして使う人は散在)
- 素の「競馬アナリスト」だと一般名詞すぎて検索で埋もれる+人間の分析者と誤認される。「AI」を頭に付けることでポジショニング(AI×全レース診断)がそのまま名前になる
- 同ニュアンスの代替案(上が優先): 「競馬AIアナリスト」「馬券AIアナリスト」「AI競馬ラボ」

@ハンドル候補(15字以内、改名時に空きを確認して上から順に):
1. `ai_keiba_lab`
2. `ai_uma_analyst`
3. `keiba_ai_report`

表示名は重複可なのでハンドルだけ空きを確認すればよい。
名前を変える場合は`src/lib/sns/theme.ts`の`SNS_BRAND.name`を直せば全画像に反映される。

## 2. 既存Xアカウント(2,500フォロワー)の転用手順

改名してもフォロワー・アカウント歴は維持される(新規開設よりスパム判定にも強く、初動有利)。

1. 設定 → アカウント情報 → ユーザー名(@)を変更
2. プロフィール編集 → 表示名を「AI競馬アナリスト」に変更
3. bio・ヘッダーを差し替え(bio文は§3、ヘッダーは`sns-out`の`digest-og.png`を1500x500にトリミングして流用可)
4. 旧ジャンルの固定ツイートを外し、§4のリニューアル告知を投稿→固定
5. 旧ジャンルの投稿は消さなくてよい(消すと不自然。流れていくに任せる)

⚠️ 旧ジャンルと競馬が無関係な場合、告知直後に多少フォロワーが減るのは正常。
2,500人の既存基盤は「初動のインプレッション源」としてそれでも新規開設より圧倒的に有利。

## 3. プロフィール(bio)文 ※160字以内

> AIが平場も重賞も週末の全レースを診断する競馬予想。オッズの妙味×回収率重視。予想は発走前に公開、結果は外れも全部報告、収支は自作システムで自動集計。目標は的中率20%・回収率115%。馬券は自己責任で。20歳未満の馬券購入は禁止されています。

(実運用実績が溜まったら「実運用n件で回収率◯%」を件数つきで追記。バックテスト値はbioに載せない)

## 4. リニューアル告知 → そのまま固定ツイート

> 【リニューアル】このアカウントは今日から、自作AIによる競馬予想の記録になります。
>
> 約束:
> 1. 予想は必ず発走前に投稿(後出しなし)
> 2. 的中も外れも全部報告
> 3. 収支は自作システムで自動集計(手集計で盛れない仕組み)
> 4. 実績が証明できるまで有料予想は売りません
>
> 平場も含めて週末の全レースをAIが診断します。まずは記録から。

※140字を超えるので、Premium未加入の場合は「約束」4行を画像化(スクショ)して添付し、本文は
「【リニューアル】今日から自作AIによる競馬予想の記録になります。約束は画像の4つ。平場も含めて週末の全レースをAIが診断します。まずは記録から。」に短縮する。

## 6. 毎週のルーティン(コマンド)

**金曜夜(土曜分の診断が済んでから)**:
```bash
npm run sns:pack -- --date <土曜の日付> --mode preview --env-file .env.local
```
→ `sns-out/<日付>-preview/`の`posts.md`を開き、「前日夜ポスト」を20:00-21:00に投稿(digest-og.png添付)

**土日朝(7:30-8:30)**: 同じpackの「当日朝ポスト」を投稿。S/A評価レースは個別ポスト(race-*-og.png)も任意で

**土日夕方(最終レース確定後、16:30以降)**:
```bash
npm run sync:netkeiba:payouts -- --date <今日> --allow-today --env-file .env.local
python3 scripts/compute_recommendation_results.py --env-file .env.local
npm run sns:pack -- --date <今日> --mode results --env-file .env.local
```
→ 「結果ポスト」を朝の予想ポストの**引用RT**で投稿(results-og.png添付)

**月曜夜**: `/dashboard`のスクショ+週次振り返り(数字は必ず件数つき)

※`npm run dev`が起動していることが前提(画像生成がdev server経由のため)

## 7. 投稿時の禁止事項(毎回チェック)

- 「絶対」「鉄板」「勝確」「必ず」→ 使わない(景表法+X規約+信頼の3点でNG)
- netkeiba・JRA等のスクショや画像の転載 → 使わない(画像は自前生成のみ)
- バックテスト値と実運用値の混同 → 数字には必ず「実運用/バックテスト」「n件」を併記
- 発走後の予想投稿(後出し) → 論外。間に合わなかったレースは黙って見送る

## 8. TikTok(8,000フォロワー)の運用

**転用**: プロフィールを競馬AI仕様に変更(bio例: 「AIが週末の競馬を全レース診断🐎 予想と結果は外れも全部X(@ハンドル)で公開中」)。X連携リンクを設定。

**投稿(週3本目安、すべてpackから自動生成済み)**:
- 金 or 土朝: `video-preview.mp4`(診断ダイジェスト→狙いレースのスライド、21秒前後)
- 土日夕方: `video-results.mp4`(結果カード6秒)or results-story.pngを**写真モード**で
- 写真モード(カルーセル)も有効: `*-story.png`をそのまま複数枚投稿(TikTokは静止画カルーセルの伸びが良い)
- キャプションはposts.mdのTikTok欄からコピペ。**音源はTikTok内のトレンド音源を付ける**(動画は無音で書き出してある。権利的にも安全で、アルゴリズム的にも有利)

**注意**: TikTokは賭博関連コンテンツの扱いがXより厳しめ。「買い目を煽る」より「AIで競馬を分析してみた」のデータ分析コンテンツとして見せる。金額・収支の強調はXに寄せ、TikTokは仕組み・結果の面白さで新規層を釣ってXへ送客する役割分担。

## 9. 実装済みの生成システム(参照)

- 画像3テンプレ: `/api/sns/race-card/[raceId]`・`/api/sns/digest?date=`・`/api/sns/results?from=&to=`(各`?format=story`で縦型1080x1920)
- 実装: `src/lib/sns/`(theme=配色・ブランド名 / font=Noto Sans JPサブセット化 / cards=テンプレJSX)
- パック生成: `scripts/sns/buildContentPack.ts`、動画: `scripts/sns/makeVideo.ts`(ffmpeg-static)
- 出力先: `sns-out/<日付>-<mode>/`(gitignore済み)
