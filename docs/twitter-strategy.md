# Twitter(X)運用プラン — 競馬予想アプリ

作成: 2026-07-16(調査+プラン策定)。実行開始は2026-07-20週から。
投稿文言・画像の中身は`/dashboard`の実績(2026-07-18以降のレースを集計、初回確定は7/20月曜バッチ後)が溜まり次第、随時更新する。

---

## 1. 調査サマリー

### 1.1 伸びている競馬予想アカウントのパターン

| アカウント | 規模 | スタイル | 無料/有料の線引き |
|---|---|---|---|
| 瀧川寿希也 | X 12.9万 | 本命・穴馬の見解+的中報告を毎日、テキスト中心 | 完全無料(知名度・実名顔出しが収益源) |
| AIウマスギ | SNS計20万超 | Xで注目馬・消し候補を無料公開、結果報告も定常運用 | **評価・注目馬は無料、買い目全体はnote有料** |
| けんしろう(note) | note 7.3万 | 追い切り重視の堅め予想、週4回前後 | 500円/1R、一部無料 |
| AiBA | 中規模 | AI予想。中央/地方でアカウントを分離 | — |

共通項:
- **直近成績を数値で明示**(「直近10戦+48,110円」「回収率138%」のように期間・件数つき)
- **的中報告を予想ツイートと対で出す**(後出しでない証明)
- 肩書き・実名・経歴などの権威付け(うちはAI+システムの透明性で代替する)
- フォロワー数≠予想力は読者側にも浸透済み。相互フォローで数だけ膨らんだ予想垢への警戒感が強い

その他の実例からの教訓:
- 投稿するだけではフォロワーは増えない(競艇予想垢の実験: 投稿のみ→フォロワー0、自分からいいね・リプ等のアクション→初めて反応が付く)
- 「1ヶ月で1万人」系は有名人リプ欄への露出などの飛び道具で、予想の信頼構築とは別物。真似しない
- **note有料販売はフォロワー基盤がないと売れない**(競馬note販売で「時給30円」に終わった失敗談あり)。「先にXで無料発信→基盤ができてから有料化」の順序は正しい

### 1.2 Xアルゴリズム(2026年時点)

- **画像・動画付き投稿はスコア約2倍**。さらに画像内の文字も機械的に読まれて評価対象になる → レース名・馬名・数字が入った情報量の多い画像は追い風
- **初動30〜60分の反応がその後の拡散を決める** → ターゲットがアクティブな時間に投稿することが最重要
- リプライ(双方向の会話)が最も重い。ただし絵文字だけ等の低品質リプは逆効果
- 動画は10秒以下/10〜60秒/60秒超で別評価軸(将来、診断画面の画面録画などを試す余地)

### 1.3 時間帯データ(日本のX一般)

- 平日: 6〜8時(通勤)、12〜13時、15〜17時、20〜22時(ゴールデンタイム)
- 土曜: 11〜13時からアクティブ増。**8〜10時は投稿が少ない「穴場」**
- 日曜: 12〜14時が中心
- 競馬文脈での解釈: 枠順確定が金・土 → **金曜夜20〜22時が「明日の予想を仕込む」時間**、**当日朝7:30〜9時が「今日どのレースを買うか決める」時間**(1Rの発走は9:35頃)。この2つがコア枠

### 1.4 法律・規約・リスク

- 予想の有料販売自体は合法(許認可不要)。ただし:
  - **「絶対当たる」「100%」等の断定・誇大表現は景品表示法/消費者契約法リスク**+X/noteの規約リスク。表現は常に「〜と判断」「妙味がある」に留める
  - netkeiba・JRA等の画像・映像・データの転載はNG。**画像は自前生成のみ**を使う(アプリのUIはこの点で最初からクリーン)
  - 未成年への賭博誘引NG。「馬券は自己責任・20歳未満は購入不可」を固定ツイート等に明記
  - 予想の盗用(他人の買い目の転売)がnote界隈で問題化した経緯あり。うちはAI自前診断なので無関係だが、逆に「盗用でない証明」として診断根拠を画像内に載せる価値がある

## 2. ポジショニング

**「平場も全レースAIが診断し、妙味のあるレースだけ本気で買う」**

- 仮説どおり、重賞中心の発信は飽和している一方、**平場を毎週・画像付き・定型フォーマットで出し続けるアカウントは手薄**。全レース系のAI予想サイトはあるが、Xで平場を「見た目が本格的な画像」で継続発信している例は目立たない
- もう一つの差別化は**透明性**: 予想は発走前に投稿(タイムスタンプ証明)、結果は外れも全部報告、集計は自作ダッシュボードで自動(手集計での盛りが構造的に不可能)。実績が薄い今こそ「これから全部記録します」と宣言して始めるのが最もクリーンで、後発の弱みを誠実さの証明に転換できる

## 3. 投稿設計

### 3.1 週間スケジュール(週10〜13投稿目安)

| 曜日 | 時刻 | 内容 |
|---|---|---|
| 金 | 20:00〜21:00 | **明日の診断ダイジェスト**(土曜分)。shutuba-watch(毎日19:00)→診断→投稿の流れと整合 |
| 土 | 7:30〜8:30 | **今日の狙いレース**(当日版、S/A中心+平場数レース)。8〜10時の「穴場」枠 |
| 土 | 16:30〜17:30 | **結果報告**。的中・外れとも、朝の予想ツイートを引用RTで対にする |
| 土 | 20:00〜21:00 | 日曜分の診断ダイジェスト |
| 日 | 7:30〜8:30 | 今日の狙いレース(当日版) |
| 日 | 16:30〜17:30 | 結果報告+**週末収支まとめ** |
| 月 | 19:00〜21:00 | **週次振り返り**(ダッシュボード画像)。月曜7:00の`compute_recommendation_results`バッチ後で数字が確定している |
| 火〜木 | 任意・週1〜2本 | 軽めの解説枠: 「AIはなぜこの馬を軸にしたか」「外れたレースの敗因分析」「システムの仕組み」など。無理に埋めない |

- 運用初期(最初の1〜2ヶ月)のKPIはフォロワー数ではなく **(a)週次サイクルを一度も落とさない (b)予想→結果→集計のループを毎週完走 (c)インプレッション/プロフィールアクセスの推移**
- 初動ブースト: 競馬垢への「データで一言添える」質の高いリプに限定して交流(絵文字だけ等の低品質リプはアルゴリズム上も逆効果)
- ハッシュタグ: 重賞はレース名タグ(#小倉記念 等)、平場は開催場タグ(#小倉競馬 #函館競馬 等)+#競馬予想。多用しない(2〜3個まで)
- X Premium(認証・長文・アナリティクス・リプ上位表示)は月額コストに見合うので加入推奨

### 3.2 画像フォーマット(3テンプレート)

サイトのダーク基調+電光掲示板風(オレンジ/シアン)をそのまま画像化し、ブランドを統一する。サイズは16:9(1200×675)基準。

1. **レース診断カード(1レース1枚)** — レース名・条件/race_rankバッジ(S=金)/本命・相手/各馬のS〜Cランク/危険な人気馬タグ/根拠の短評(race_rank_reason要約)。「手が込んでそう」の主役。画像内テキストはアルゴリズムにも読まれるため、馬名・レース名を画像に焼き込むこと自体がリーチに効く
2. **当日ダイジェスト(1日1枚)** — 今日の診断対象レース一覧をランク付きで俯瞰。「平場も全部見ている」ことが一目で伝わる、差別化の核
3. **結果・収支カード** — 的中◯/購入◯件、投資→払戻、ROI(実測)。`/dashboard`の数字と連動し、**必ず件数(n)を併記**

実装メモ(来週以降の実装タスク、優先度順):
- フェーズ0(即日可): レース詳細ページ・ダッシュボードの手動スクリーンショットで開始
- フェーズ1: Next.jsの`ImageResponse`(@vercel/og系)でシェア画像ルート(`/api/races/[raceId]/card`等)を新設し、上記3テンプレを自動生成。OGPにも流用してサイト流入時の見栄えを揃える
- フェーズ2(任意): 投稿自体の自動化。X APIの無料枠(執筆時点で月500投稿程度、要確認)で週10〜13投稿は収まる想定。ただし当面は手動投稿で文面の手応えを掴む方が先

### 3.3 無料/有料の線引き

当面(実運用実績が固まるまで)は**全部無料**。将来の有料化は次の構図:

- **無料**: race_rank、各馬のS〜Cランク、本命、危険な人気馬、根拠の要約 — AIウマスギ型「評価は無料、買い目は有料」と同じ構図が、既にアプリのデータ構造(ランク/買い目の分離)と一致している
- **有料**: 買い目の全体(券種・相手・金額配分)+フル分析(premium/Opus診断の中身)

⚠️ **「race_rank=S限定を有料化」案への注意**: 2026-07-14の分析で、race_rankは的中率とほぼ無相関、的中はA評価に集中しS評価からの的中はゼロ(nは小さい)という結果が出ている。「S=売り物」と固定すると、有料予想の成績が無料分より悪いという最悪の構図になり得る。**有料の単位は「rank」ではなく「本気診断(premium)を実行したレース」とし、どのセグメントを売るかは実運用データ(ランク別ROI)が溜まってから決める。**

- 価格相場: note単品300〜900円/R が中心帯。月額マガジンも可。手数料は決済5%+プラットフォーム10%。**300円は「安すぎて価値を疑われる」失敗実例があるため500〜900円寄りにする**
- **有料化のゲート(2026-07-17に訂正)**: (a)`/dashboard`の実運用集計でn≥50件 (b)売る対象セグメントのROI>100%を継続的に提示できる (c)note無料記事にフォロワー以外の流入がある。**⚠️当初あった「フォロワー1,000〜」の条件は撤回した** — [sns-growth-research.md](sns-growth-research.md)の調査で、フォロワー数と売上がほぼ無関係(13人で初売上/6,000人で5部、購入者の大多数はフォロワー以外)と判明したため。フォロワーを待つと改善サイクルの開始が遅れるだけになる。実績ゲートは集客の条件ではなく**信頼の条件**なので維持する
- 「実績が証明できるまで売りません」は固定ツイートで公言してよい(強い信頼シグナル)

### 3.4 プロフィール・固定ツイート(案)

**名前の方向性**: 「AI」+「妙味/回収率」+覚えやすい馬関連語の組み合わせ。サイトの電光掲示板の世界観と揃える(例の方向性: 「妙味AI」「オッズボードAI」系。最終決定はユーザー)

**プロフィール文(案、160字以内)**:

> AIが平場も重賞も週末の全レースを診断する競馬予想。オッズの妙味×回収率重視。予想は発走前に公開、結果は外れも全部報告、収支は自作システムで自動集計。目標は的中率20%・回収率115%。馬券は自己責任で。20歳未満の馬券購入は禁止されています。

(実績数値はプロフィールには当面書かない。書けるようになったら「実運用n件で回収率◯%」を件数つきで追記)

**固定ツイート(案)**:

> 【このアカウントの約束】
> 1. 予想は必ず発走前に投稿します(後出しなし)
> 2. 結果は的中も外れも全部報告します
> 3. 収支は自作システムで自動集計します(手集計で盛れない仕組み)
> 4. 数字には必ず件数を添えます
> 5. 実績が証明できるまで有料予想は売りません
>
> AIが平場も含めて週末の全レースを診断しています。まずは記録を積むところから。

## 4. 誇張せず信頼を積み上げる原則

1. **発走前投稿+引用RTでの結果報告**をワンセットの型にする(タイムスタンプが唯一の証明手段)
2. **外れ報告を的中報告と同じテンションで出す**。的中だけの報告は読者が最も警戒するパターンだと調査でも確認済み
3. 数字は**実測値・期間・件数をセット**でのみ出す。「回収率105.6%(バックテスト累計175件)」のように。バックテストと実運用は必ず区別して表記し、**当面「回収率◯◯%」を看板(プロフィール・固定)に掲げない**
4. 断定・煽り表現を使わない(景表法・規約・信頼の3点でNG)。「絶対」「鉄板」「勝確」→「妙味があると判断」「期待値が高い」
5. 目標未達も開示する。「目標: 的中率20%/ROI115%、現状: 13.1%/105.6%(n=175)」の形式は、達成済みを装うより長期的に強い
6. 免責(自己責任・20歳未満購入不可)を常設
